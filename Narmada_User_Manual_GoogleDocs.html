1) System Purpose and Role Matrix

The portal is a river pollution and water quality analysis platform with geospatial visualization and model execution workflows.

Role Main Objective Can Upload? Can Run Model Pipeline? Can View Outputs?
Admin Central data administration, interpolation generation, and system model runs Yes (Display, Training, Testing) Yes (/run) Yes
Collaborator Field and partner data contribution with account-specific workspace Yes (Display, Training, Testing) Yes (/runcol) Yes
Viewer Read-only access through OTP for map analytics and modelling output No No Yes

5) Admin Upload SOP (Step-by-Step)

5.1 Admin Upload Display

  1. Login as admin.
  2. Open Upload Display Data page.
  3. For CSV update, provide Year, Month, and CSV file.
  4. For raster update, choose LULC .tif and/or POP .tif.
  5. Click Upload and Process.
  6. Use Clear Directory button only when reset is required.

5.2 Admin Upload Training

  1. Open Admin Modelling page.
  2. Open Upload Training Data.
  3. Upload Year + Month + CSV for monthly updates.
  4. Optionally upload LULC and POP .tif files.
  5. Validate status message for success.

5.3 Admin Upload Testing and Pipeline Run

  1. Open Upload Testing Data.
  2. Upload one or more CSV files.
  3. Upload LULC and POP year rasters.
  4. Click Upload and Process.
  5. System triggers model sequence automatically and redirects to output page on success.

6) Collaborator Upload SOP (Step-by-Step)

6.1 Collaborator Upload Display

  1. Login as collaborator.
  2. Open Upload Display page.
  3. Upload GeoJSON layers in the designated fields.
  4. Upload raster files and CSV folders.
  5. Wait for chunk progress to finish before leaving page.

6.2 Collaborator Upload Training

  1. Open Upload Training page.
  2. Upload precip, temp, streamflow, waterlevel folders.
  3. Upload LULC/POP raster folders.
  4. Wait for complete chunk upload confirmation.

6.3 Collaborator Upload Testing and Pipeline Run

  1. Open Upload Testing page.
  2. Upload testing CSV files and LULC/POP rasters.
  3. Click Upload and Process.
  4. System calls /runcol and redirects to collaborator output on success.

7) Exact File Formats, Naming Rules, and Data Templates

7.1 CSV format for Admin display/training update APIs

Used by:

  • /api/admin-update-stations-display
  • /api/admin-update-stations-training-data

Mandatory columns:

  • station
  • one or more dataset columns (precip, temp, streamflow, waterlevel, maxt, mint, wqi, etc.)

Template:

station,precip,temp,streamflow,waterlevel,maxt,mint,wqi
74.456489_22.039138,79.08,26.79,165.38,190.94,33.25,22.06,43.41
74.890280_22.079645,70.05,30.46,160.47,236.86,33.19,21.81,35.65

Month handling: UI month can be Jan..Dec or numeric 1..12.

7.2 Storage format generated by update APIs

Per station and per variable files are maintained as monthly table:

Year,Jan,Feb,Mar,Apr,May,Jun,Jul,Aug,Sep,Oct,Nov,Dec
2023,47.00,49.83,46.67,43.51,43.81,44.11,57.88,71.82,53.95,36.08,39.58,42.91
2024,42.05,41.02,39.98,39.12,40.29,41.46,,,,,,

7.3 Testing CSV format for GNN testing (Admin and Collaborator)

Mandatory columns (exact):

station,streamflow,precip,waterlevel,maxt,mint,wqi

Example:

station,streamflow,precip,waterlevel,maxt,mint,wqi
74.456489_22.039138,165.38,79.07,190.94,33.24,22.06,43.41

7.4 Testing CSV filename rule (critical)

Filename must include period token:

YYYY_Mon

Valid examples:

  • 2024_Aug.csv
  • 2024_Oct.csv
  • 1776166885_2024_Aug.csv

Invalid examples:

  • Aug_2024.csv
  • 2024-08.csv
  • test.csv (no period token)

7.5 Station naming rule inside testing CSV

Testing script parses station as numeric lon_lat via underscore split.

Valid:

  • 74.456489_22.039138

Invalid for parser:

  • Station_74.5_22.5
  • Barmanghat

10) Troubleshooting Playbooks

10.1 Upload succeeds, output missing

  1. Confirm file landed in expected role folder.
  2. Check naming convention (especially testing CSV and year rasters).
  3. Confirm required columns exist.
  4. Re-run upload and monitor UI status.

10.2 Pipeline fails after testing upload

  1. Check testing CSV column schema.
  2. Check station values are lon_lat format.
  3. Verify testing_input/lulc and testing_input/pop have YEAR.tif files for all years in CSV names.
  4. Ensure torch environment is available to pipeline.

10.3 Map layer not visible

  1. Enable layer checkbox/toggle.
  2. Confirm data file exists in expected geojson/raster path.
  3. Refresh map page after upload completion.

11) Data Governance and Safety Notes

  • Do not upload mixed period files in the same testing run unless year-tokened filenames and raster support are complete.
  • Use Clear Directory operations with care; they remove all files in target folder.
  • Maintain backup before mass re-upload.
Security hardening required before production: credentials, SMTP password, and database URI should be moved from source code to environment variables.

13) Appendix B - Operator Checklists

13.1 Pre-upload checklist

  1. Role selected correctly (Admin vs Collaborator).
  2. Extension correct (.csv, .tif, .geojson).
  3. Required columns present.
  4. Testing CSV filename has YYYY_Mon token.
  5. LULC and POP test rasters named YEAR.tif.

13.2 Post-upload checklist

  1. Status message shows success.
  2. Destination files exist in expected directory.
  3. For testing run, output page opens successfully.
  4. Map layers load for selected year/month.
